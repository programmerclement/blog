# Flutter for Web Developers

As a web developer who spent years building with React and Node.js, diving into Flutter was both exciting and challenging. In this guide, I'll share everything I learned making the transition.

## Why Flutter?

Flutter lets you build natively compiled apps for mobile, web, and desktop from a single codebase. For a web developer, the appeal is obvious: learn one framework, target every platform.

**Key advantages:**
- Single codebase for Android, iOS, Web, and Desktop
- Fast development with Hot Reload
- Beautiful, consistent UI across platforms
- Growing ecosystem and strong Google backing
- Dart language is easy to learn for JavaScript developers

## Setting Up Flutter

```bash
# Download Flutter SDK from flutter.dev
# Then add to PATH and run:
flutter doctor

# Create a new project
flutter create my_first_app
cd my_first_app
flutter run
```

## Dart vs JavaScript

As a JS developer, you'll find Dart familiar:

```dart
// JavaScript
const greet = (name) => `Hello, ${name}!`
const numbers = [1, 2, 3].map(n => n * 2)

// Dart
String greet(String name) => 'Hello, $name!';
final numbers = [1, 2, 3].map((n) => n * 2).toList();
```

Key differences:
- Dart is **strongly typed** (similar to TypeScript)
- `null safety` built in (no more `undefined is not a function`!)
- `final` for immutable variables, `var` for mutable
- No `undefined` — only `null`

## Flutter Widgets = React Components

In Flutter, everything is a widget — just like React components:

```dart
// React way
function Button({ label, onPress }) {
  return <button onClick={onPress}>{label}</button>
}

// Flutter way
class CustomButton extends StatelessWidget {
  final String label;
  final VoidCallback onPress;
  
  const CustomButton({required this.label, required this.onPress});
  
  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: onPress,
      child: Text(label),
    );
  }
}
```

## Layout in Flutter

Flutter uses a box model similar to CSS Flexbox:

```dart
Column(  // Vertical stack = flex-direction: column
  children: [
    Row(  // Horizontal = flex-direction: row
      mainAxisAlignment: MainAxisAlignment.spaceBetween,
      children: [
        Text('Left'),
        Text('Right'),
      ],
    ),
    Padding(
      padding: EdgeInsets.all(16),
      child: Text('Padded content'),
    ),
    Expanded(  // flex: 1
      child: Container(color: Colors.blue),
    ),
  ],
)
```

## State Management

For simple state, use `setState`:

```dart
class CounterPage extends StatefulWidget {
  @override
  _CounterPageState createState() => _CounterPageState();
}

class _CounterPageState extends State<CounterPage> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text('Count: $_count', style: TextStyle(fontSize: 24)),
            ElevatedButton(
              onPressed: () => setState(() => _count++),
              child: Text('Increment'),
            ),
          ],
        ),
      ),
    );
  }
}
```

For complex apps, use **Provider**, **Riverpod**, or **BLoC** — similar to Context API or Redux.

## Making HTTP Requests

```dart
import 'package:http/http.dart' as http;
import 'dart:convert';

Future<List<Post>> fetchPosts() async {
  final response = await http.get(
    Uri.parse('https://api.example.com/posts'),
    headers: {'Authorization': 'Bearer $token'},
  );
  
  if (response.statusCode == 200) {
    final List<dynamic> data = json.decode(response.body);
    return data.map((json) => Post.fromJson(json)).toList();
  } else {
    throw Exception('Failed to load posts');
  }
}
```

## Building for Production

```bash
# Build Android APK
flutter build apk --release

# Build iOS (requires Mac + Xcode)
flutter build ios --release

# Build for web
flutter build web

# Build app bundle for Play Store
flutter build appbundle
```

## My Lessons Learned

After building three Flutter apps (including an Android AI assistant with OCR capabilities), here's what I wish I knew:

1. **Hot reload is magical** — changes reflect in milliseconds
2. **Everything is a widget** — embrace this mental model
3. **Dart is not scary** — if you know TypeScript, you'll be comfortable in days
4. **Use packages from pub.dev** — the ecosystem is rich
5. **Layout takes getting used to** — Column/Row/Stack behave differently from CSS

## Conclusion

Flutter is an incredible framework that has genuinely changed how I think about cross-platform development. Coming from React, the widget tree feels natural, and Dart is a joy to write once you get past the initial learning curve.

My advice: build something. Start with a simple todo app, then a weather app, then something real. You'll be shipping to the Play Store in no time.
